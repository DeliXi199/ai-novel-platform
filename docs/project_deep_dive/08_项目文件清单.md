## 4. 文件清单怎么用最省时间

### 如果你要看“业务入口”
直接看 `api/routes/`。

### 如果你要看“状态长什么样”
直接看 `models/`、`schemas/`、`story_runtime_support.py`、`story_architecture.py`。

### 如果你要看“下一章怎么生成”
直接看 `chapter_generation_*` 系列。

### 如果你要看“为什么不会乱改前文”
直接看 `hard_fact_guard*`、`story_fact_ledger.py`。

### 如果你要看“前端为什么能显示那么多状态”
直接看 `story-studio` / `story-workspace` / `live-runtime` 接口，以及 `story_architecture.build_story_workspace_snapshot()`。

## 5. 关于测试目录的一句提醒

`backend/tests/` 数量不少。  
这通常说明项目经历过多轮“写着写着状态回归出问题”的阶段，所以作者已经开始用测试保护：

- 近纲；
- 连续性；
- 标题；
- prompt；
- TTS；
- 任务系统；
- Story Bible V2 地基；
- hard fact guard；
- 双阶段生成。

对这种项目来说，测试不是装饰品，而是防止叙事状态塌房的脚手架。
